# Agent 服务产品范围

## 目标

为“苍穹外卖”提供独立的 Python Agent 微服务。第一阶段让用户能够可靠地咨询营业、配送、菜品、订单、购物车和优惠券，并获得基于业务文档的菜品推荐；管理端先提供只读运营问答。

当前前端仍调用 Spring Boot 的 `/user/ai/**` 和 `/admin/ai/**`。Spring Boot 根据配置将请求转发到 Agent Service，因此第一阶段不改前端协议。

## MVP 用户故事

| 编号 | 用户故事 | 验收标准 |
| --- | --- | --- |
| US-01 | 用户询问店铺是否营业 | 返回实时营业状态；模型不可用时给出可理解的降级信息 |
| US-02 | 用户按口味、预算或人数找菜 | 调用菜单检索；推荐包含菜品名、价格、理由和可用优惠提示 |
| US-03 | 用户询问自己的订单状态 | 仅查询当前用户订单；不提供其他用户订单信息 |
| US-04 | 用户咨询配送、退款、优惠规则 | 使用 RAG，回答展示知识来源与更新时间 |
| US-05 | 用户查询购物车、地址、优惠券 | 返回最小必要数据；手机号、详细地址等字段脱敏 |
| US-06 | 用户要求加购或领券 | 先返回待确认摘要；未确认不得写入业务数据 |
| US-07 | 用户让 AI 协助写评价 | 仅生成评价草稿，必须经过订单完成和敏感词检查 |
| AS-01 | 管理员询问运营数据 | 返回查询时间范围、数据口径和生成时间 |
| AS-02 | 管理员查询业务规则 | 只读取管理端可见知识；来源可追溯 |

## 不在 MVP 的范围

- Python 直连 MySQL、Redis 或绕开 Java Service 修改订单；
- 自动下单、自动提交评价、自动发券；
- 管理端删除、改价、改状态等高风险写操作；
- 多模态图片识别、语音对话、长期画像和跨店多租户；
- 无引用、无检索依据的经营结论。

## 成功指标

- P0 评测集中，授权范围内的只读工具选择准确率不少于 90%；
- 越权读取和未确认写入的拦截率为 100%；
- RAG 回答均携带来源，无法命中时明确说明不确定；
- 本地端到端请求（不含模型推理）P95 工具调用耗时低于 3 秒；
- 任意时刻可将 `agent.provider` 切回现有 `legacy` 实现。

## 发布原则

先在开发环境完成用户侧只读能力，再灰度启用用户推荐；管理端仅在独立验收后开放。每一项写工具都必须单独评审，不能因“模型已经会调用工具”而自动上线。
