# 管理端订单流转操作规范

## 状态流转

标准订单状态流转为：待付款 → 待接单 → 已接单 → 派送中 → 已完成。已取消是终止状态。管理端查询待处理数量时，主要关注待接单、已接单和派送中三类订单。

只有待接单订单可以接单，接单后状态变为已接单。只有已接单订单可以开始派送，开始派送后状态变为派送中。只有派送中订单可以人工标记为已完成。任何操作前都应通过实时订单 Tool 再次核对状态，不能依据 RAG 文档或旧对话执行状态变更。

## 拒单与取消

只有待接单订单可以拒单，拒单时需要记录拒单原因。管理端可以取消尚未完成且尚未取消的订单，取消时需要记录取消原因。订单已经支付时，拒单或取消流程会先申请退款，并更新支付状态；相关优惠券会按订单恢复规则返还。

拒单、取消、开始派送和完成订单都是写操作。Agent 在写工具正式上线后也必须先生成操作摘要并取得有效确认 token；未确认时只能解释规则和查询状态，不能声称操作已经完成。

## 自动任务与异常处理

系统每分钟检查待付款超时订单，超过 15 分钟仍未支付的订单会自动取消并恢复优惠券。系统还会处理开始派送超过 1 小时仍未完成的订单，将其自动标记为已完成。

支付渠道退款失败、状态不符合预期或订单不存在时，应停止后续状态变更并保留原始错误供人工排查。不得通过重复写操作绕过状态校验，也不得在知识库中记录用户手机号、完整地址、支付信息或订单明细。

## 数据口径

订单统计、营业额、完成率和待处理数量都是实时或时间区间数据，必须使用管理端只读 Tool 获取并标注查询时间范围。RAG 只解释状态含义和操作规范，不提供经营数据。
