# 管理端菜单与套餐维护 SOP

## 适用范围

适用于管理员查询、修改门店菜单、菜品、规格、套餐和售罄状态。Agent 只能生成结构化预览并请求管理员确认，不能直接执行任意字段更新。

## 修改前检查

1. 确认管理员角色、门店范围和租户范围。
2. 查询目标记录当前版本、当前价格、当前售卖状态和关联套餐。
3. 校验新价格为非负金额，售卖时间合法，必填配料和过敏原信息完整。
4. 检查是否影响已经创建但未完成的订单。
5. 生成旧值/新值、影响数量、风险等级和回滚说明。
6. 通过 `interrupt()` 展示确认卡片，确认后才调用写 Tool。

## 高风险变更

价格大幅变化、批量上下架、修改过敏原、修改已售订单关联菜品、物理删除和跨门店操作属于高风险变更。默认需要二次审批或转人工，Agent 不应自行绕过审批。

## 失败处理

- 版本冲突：重新查询并生成新预览，不覆盖他人修改。
- 权限不足：立即拒绝并记录安全事件。
- Java 返回未知结果：查询最终状态，不重复提交。
- 部分成功：返回逐项结果和审计号，进入人工复核。
