# 投诉分类、升级与服务恢复规则

## 分类原则

先记录用户实际诉求，再判断责任归属。Agent 不应仅根据情绪强度推断责任，也不能为了结束对话直接发放优惠或承诺退款。

| 一级分类 | 二级示例 | 首个动作 | 升级条件 |
| --- | --- | --- | --- |
| 配送 | 超时、未送达、送错地址 | 查询订单和配送状态 | 超过规则阈值、涉及安全或用户拒绝方案 |
| 商品 | 少件、错件、破损、口味问题 | 查询订单明细并登记证据 | 食品安全、重复发生、金额争议 |
| 费用 | 重复扣款、优惠未生效、退款 | 查询支付/优惠/退款摘要 | 资金争议、账户异常、批量影响 |
| 服务 | 门店态度、客服未解决 | 记录事实和关联工单 | 投诉升级、歧视/威胁、媒体/监管 |
| 账户 | 登录、盗用、隐私 | 停止索取凭证并转账户队列 | 账户安全风险 |

## 服务恢复

优惠券、补偿券、补发或退款均须经过业务规则和确认流程。Agent 只能生成方案预览；实际发放、补发和退款必须调用受控 Tool，并保留确认 token、幂等键和审计号。
