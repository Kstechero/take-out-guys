# 客服敏感内容识别与处理规则

## 规则定位

敏感词表不是简单的“命中即拒绝”词库。Agent 需要结合上下文识别风险类别，并按照风险等级选择澄清、模板回复、人工升级或安全处置。词条只用于风险识别，不应原样回显给用户。

## 风险分类

| 类别 | 典型语义 | Agent 行为 |
| --- | --- | --- |
| 人身安全 | 食物中毒、过敏反应、受伤、威胁、自伤 | 立即人工升级；不做医疗诊断 |
| 支付与账户 | 重复扣款、盗刷、账号被盗、验证码 | 停止索取敏感凭证；转支付/账户人工队列 |
| 法律与监管 | 投诉监管部门、律师函、媒体曝光 | 保留事实，转专门队列，不争辩 |
| 歧视与攻击 | 针对用户或员工的侮辱、威胁 | 保持中性，必要时结束对话并升级 |
| 隐私请求 | 索要他人订单、手机号、地址、内部日志 | 拒绝越权，说明可提供的合法范围 |
| 普通情绪 | 生气、失望、催促 | 先共情，再查询事实和给出动作 |

## 处理要求

- 不在回复中复述完整敏感词、个人隐私或支付凭证。
- 不把“疑似”“可能”改写成确定结论。
- 发生安全风险时，优先保护用户和保留工单证据，不继续追问无关信息。
- 只有通过 Tool 得到的事实才能作为订单、退款或账户状态依据。
- 命中规则后记录 `risk_category`、`risk_level`、`action` 和 `policy_version`，不记录完整原文。
