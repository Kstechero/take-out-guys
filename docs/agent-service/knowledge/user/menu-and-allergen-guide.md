# 食品安全与过敏原回答规则

## 1. 菜品事实边界

本文只说明食品安全和过敏原回答规则，不保存菜品主数据。菜品名称、配料、规格、价格和库存必须以数据库及 `menu_search`/过敏原 Tool 返回结果为准。

- 每个菜品需要展示名称、份量、基础价格、可选规格、主要配料和过敏原提示。
- 标记为“售罄”的菜品不能推荐为可下单项；可以解释售罄并建议相似菜品。
- 价格、可售规格和可用优惠以当前门店查询结果为准，不引用本文中的模拟示例价格。
- 组合套餐必须说明套餐包含项、可替换项、替换加价和不可替换项。

## 2. 过敏原回答规范

用户明确表达过敏时，Agent 必须先确认过敏原和目标菜品，再调用菜单/过敏原 Tool。不能承诺“绝对无交叉污染”。

推荐回答结构：

1. 说明已知配料中是否包含用户提到的过敏原；
2. 说明厨房共用设备或生产线可能产生交叉接触风险；
3. 建议用户在下单前向门店人工确认；
4. 如果无法确认，明确说“当前资料不足”，不要猜测。

## 3. 模拟菜品知识示例

| 菜品 | 主要配料 | 可能过敏原 | 用户侧建议 |
| --- | --- | --- | --- |
| 黑椒鸡腿饭 | 鸡腿、米饭、黑椒汁 | 大豆、小麦 | 可查询是否可去酱；不能保证无交叉接触 |
| 芝士牛肉汉堡 | 牛肉、奶酪、面包 | 乳制品、小麦、芝麻 | 乳制品过敏时不要推荐去芝士后直接下单 |
| 花生酱酸奶杯 | 酸奶、花生酱、燕麦 | 乳制品、花生、燕麦 | 花生过敏不可推荐 |

以上菜品仅为测试语料，实际下单前必须查询实时菜单和过敏原数据。
