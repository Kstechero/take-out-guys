# 下单与配送规则

## 下单条件与金额

下单时必须选择当前登录用户自己的有效收货地址，购物车不能为空。订单商品金额由服务端根据当前购物车重新计算，不采用客户端自行填写的金额。当前系统在购物车商品总额基础上增加 6 元配送费，再扣减满足条件的优惠券金额；最终应付金额不会低于 0 元。

下单成功后，系统生成订单和订单明细，并清空当前用户购物车。具体菜品价格、是否在售、购物车内容和可用优惠券均属于实时数据，回答这些问题时应查询对应 Tool。

## 订单状态

订单依次可能处于：待付款、待接单、已接单、派送中、已完成或已取消。支付成功后由待付款进入待接单；商家接单后进入已接单；开始配送后进入派送中；送达后进入已完成。

待付款订单超过 15 分钟仍未支付会被系统自动取消，订单占用的优惠券会恢复。用户询问某一订单当前状态、预计送达时间或取消原因时，必须查询订单 Tool，不能只根据本规则判断。

## 催单与配送范围

用户只能催促自己的待接单、已接单或派送中订单，其他状态不能催单。催单会向管理端发送提醒，但不代表订单状态已经变化。

当前后端没有配置可供知识库承诺的固定配送半径或区域白名单，因此不能回答“某地址一定能送达”。系统下单校验目前只确认收货地址存在且属于当前用户；具体地址是否可服务如有疑问，应建议联系人工客服，不能虚构配送范围。

派送中订单开始配送超过 1 小时仍未人工完成时，系统定时任务会将其自动标记为已完成。某个订单是否已触发该规则仍以订单 Tool 的实时结果为准。
