# 用户客服响应与工单处理 SOP

## 适用范围

用于用户 Agent 处理咨询、催单、配送异常、少件错件、退款咨询、投诉和人工转接。本文只定义处理规则和回复边界，不提供订单、菜品、库存或退款金额等实时事实。

## 标准处理顺序

1. 识别问题类型和用户诉求，不先承诺结果。
2. 需要实时事实时调用对应 Tool：订单状态、配送轨迹、订单明细、退款状态等。
3. 判断是否存在敏感内容、食品安全风险、支付风险或投诉升级条件。
4. 使用已批准模板组织回答，引用规则来源和下一步动作。
5. 需要人工处理时创建工单并告知用户预计响应方式，不编造处理时限。

## 回复边界

| 场景 | 可以说明 | 不可以说明 |
| --- | --- | --- |
| 催单 | 查询当前订单状态、解释状态含义、说明下一步 | 未查询就承诺具体送达时间 |
| 退款咨询 | 解释规则、查询退款状态、创建退款预览 | 直接承诺最终退款金额或到账时间 |
| 少件/错件 | 核对订单明细、登记售后、请求必要证据 | 未核验就认定门店或配送员责任 |
| 食品安全 | 立即升级、保留证据、建议联系人工 | 诊断疾病或保证食品绝对安全 |
| 过敏原 | 查询已维护的过敏原提示、说明交叉接触风险 | 承诺“完全无过敏原” |

## 必须转人工

重复扣款、疑似过敏反应、食品异物、用户人身安全、退款争议、法律/监管投诉、媒体或平台投诉、账户盗用、批量订单异常，以及用户连续两次表示未解决的问题。

## 回复记录

记录 `intent`、`order_id` 是否已通过 Tool 核验、引用的 `policy_version`、是否触发敏感词、是否升级工单和 `trace_id`。日志不得记录完整手机号、地址、支付信息、密码、token 或完整对话原文。
