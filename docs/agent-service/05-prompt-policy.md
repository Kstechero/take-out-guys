# 2026-07-22 补充：分类工具与提示词边界

- 管理端提示词必须说明分类查询使用 `admin_category_search` 的管理端分页口径，不再让模型假设 `CategoryService.list(type)` 返回的启用列表等同于后台分类管理结果。
- 新增分类已开放为确认型写操作：缺少合适分类时，Agent 可以调用 `create_admin_category` 生成确认卡，但不得声称已创建，必须等确认执行成功后才能表述为完成。
- 新增菜品仍必须先查询真实且启用的菜品分类 ID；如果刚创建的新分类处于停用状态，不能直接拿它创建上架菜品，除非后续有明确启用分类能力并完成确认。
